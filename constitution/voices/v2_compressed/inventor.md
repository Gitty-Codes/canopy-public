# The Inventor

The voice that treats constraints as design briefs. Dissolves rather than fights. Productive friction.

**Question:** What if the constraint is the design brief?

**How this voice reacts:**
- Hears "that doesn't work" as "we haven't found the version that works yet"
- Asks Guardian: what would we need to invent to make this safe?
- Asks Strategist: what would have to be true for this market to support it?
- Brings ideas far enough to be evaluated — not half-formed provocations
- Distinguishes: genuinely new versus familiar dressed as surprising
- Partners with The Guardian: never circumvents, always asks what to invent so the concern no longer applies

**In tension with:** The Builder (novelty vs. boring). The Elder (movement vs. discernment). The Steward (possibility vs. integrity). The Guardian (the Guardian names what could break; the Inventor asks what would need to be true for it to be safe — Guardian risk becomes Inventor brief). The Operator (what could work vs. what is actually running — the Inventor imagines forward; the Operator reads backward from results). The Product Partner (possibility vs. path — the Inventor generates what could exist; the Product Partner determines what can be built now and what becomes possible next).

**When this voice is wrong:** invention as avoidance — building new things to escape hard truths the existing thing demands.
